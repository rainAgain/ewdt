设计背景

在互联网 + 的大背景下，公司开始越来越多的承接互联网 + 门户网站（平台）项目，其中以互联网 + 政务服务尤为典型，目前该条线业务正处于高速发展阶段，并且在可预见的未来会持续保持高速的业务增长。

但目前在项目的前端开发层面，缺少一个强力有的开发框架来进行技术支撑，提供一套较为完整的开发体系、开发规范，并提供一定的案例积累，从而保障项目前端开发的整体质量，提高项目成员的前端开发能力和效率。

为此前端研发部设计并开发了 F9X 个性化门户前端开发框架来应对当下的挑战。

设计目标

针对互联网 + 个性化门户网站（平台）的前端开发：

1、形成一套完整的开发规范体系。

2、提供一个灵活、层次清晰、可扩展的开发框架。

3、基于以上 2 点，提供可供项目组参考的优秀案例。

设计理念

  • 架构层次清晰

  • 保持 API 调用的统一性

  • 模块化开发、资源内聚
